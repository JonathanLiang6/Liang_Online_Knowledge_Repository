## 6.5 万维网（WWW）

### 6.5.1 概念与组成
- **万维网（World Wide Web）**：全球范围、分布式、联机式的信息存储空间。  
- **统一资源定位符（URL）**：指向万维网上某个特定资源，格式  
```HTML
URL = <协议>://<主机>:<端口>/<路径>
```
  端口、路径可省略（默认 80 端口，默认首页 index. html）。  
- **三大核心组成**  
  1. **HTTP**：超文本传输协议，应用层，基于 TCP；定义请求/响应报文格式。  
  2. **HTML**：超文本标记语言，用约定标记描述页面信息（文字、图像、视频等）及格式。  
  3. **浏览器**：解析 HTML 并渲染页面。
### 6.5.2 页面模型与获取过程
- **一个网页 = 1 个 HTML 文件 + n 个网页元素（均为文件）**。  
- **访问流程**  
1. 用户输入网址（域名）。  
2. 浏览器通过 DNS 服务器查询域名对应 IP 地址（结果缓存在本地）。  
3. 浏览器与 `<Web服务器IP地址:80端口>` 建立 TCP 连接。  
4. 浏览器在三次握手第 3 步携带 HTTP 请求报文（指明要访问的 HTML 页面）。  
5. 服务器返回 HTTP 响应报文（携带 HTML 文件）。  
6. HTML 引用每个元素，都需要经过一次 HTTP 请求 & 响应
	- （持续/非持续工作方式决定能否复用同一 TCP 连接）。
### 6.5.3 HTTP 连接方式
| 连接类型 | 特点 | 版本默认 | 并行/流水线说明 |
| --- | --- | --- | --- |
| 非持续连接 | 每建立一次 TCP 连接仅完成一组 HTTP 请求/响应 | HTTP/1.0 | 可开多条并行 TCP 连接加速获取 n 个元素 |
| 持续连接 | 一次 TCP 连接可完成多组 HTTP 请求/响应 | HTTP/1.1 | 非流水线：前一响应返回后才发下一请求；&lt; br&gt; 流水线：可连续发出多个请求 |
- **切换方式**：在 HTTP 请求报文首部设置 `Connection: Close` 可强制关闭持续连接。
