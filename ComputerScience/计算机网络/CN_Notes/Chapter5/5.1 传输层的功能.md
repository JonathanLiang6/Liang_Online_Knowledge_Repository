### 5.1.1 端口的作用
- 通过“端口号”标识本主机的一个特定进程。  
- **注意**：每台主机的端口号相互独立；TCP 与 UDP 两种协议的端口号也相互独立。  
- TCP 或 UDP 通过**套接字（Socket）= {IP 地址 : 端口号}**，唯一标识网络中一台主机上的一个应用进程。
### 5.1.2 端口号的分类
| 类别    | 范围            | 用途                         |
| ----- | ------------- | -------------------------- |
| 熟知端口号 | 0 ~ 1023      | 只能分配给被熟知的重要应用程序，开发的时候一般不使用 |
| 登记端口号 | 1024 ~ 49151  | 服务器使用，需注册                  |
| 短暂端口号 | 49152 ~ 65535 | 客户端临时使用                    |
### 5.1.3 传输层功能
1. **实现端到端（进程到进程）的通信**。  
2. **复用与分用**  
   - 复用（从上到下）：同一台主机上的多个进程可使用同一个传输层协议发送数据。  
   - 分用（从下到上）：传输层把接收到的数据正确交付到目的进程。  
3. **差错检测**  
   - TCP：检测出差错后丢弃数据，并通知发送方重传。  
   - UDP：检测出错误后直接丢弃数据，且不通知发送方。
### 5.1.4 向应用层提供的两种服务
| 服务类型    | 简介      | 特点                    |
| ------- | ------- | --------------------- |
| **TCP** | 面向连接、可靠 | 确保数据正确、完整，但开销大、实时性较差。 |
| **UDP** | 无连接、不可靠 | 数据可能出错/丢失，但速度快、开销小。   |
