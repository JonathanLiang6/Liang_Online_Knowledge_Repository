### 4.3.1 基本表示格式
- **长度**：128 bit，共 16 Byte  
- **默认写法**：冒号十六进制记法  
  - 每 16 bit 为一段，共 8 段；段间用冒号 `:` 分隔  
  - 每段写成 4 个十六进制字符，不足 4 位在前面补 0  
  - e.g `2001:0db8:0000:0000:0000:ff00:0042:8329`
### 4.3.2 压缩规则
1. **前导零省略**：每段内部的前导 0 可省，但至少保留 1 位。  
`e.g 2001:db8:0:0:0:ff00:42:8329`
2. **双冒号替换**：连续的全 0 段可用 `::` 一次性替换，**一个地址只能用一次**。  
`e.g 2001:db8::ff00:42:8329（展开后2001:0db8:0000:0000:0000:ff00:0042:8329）`
### 4.3.3 地址结构模型
一台主机的 IPv6 地址可看作：$【n位前缀 : 128-n位接口标识符】$  
- **前缀**：用于路由与子网划分，长度 n 由 CIDR 表示法给出，如 `/64`  
- **接口标识符**：通常 64 bit，可由 MAC 地址等自动生成（EUI-64）
- **IPv6 技术优点**：
	-IPv6 因为地址空间很大，所以可以**划分为更多的层次**
	- IPv6 支持**即插即用** (IP 地址自动配置): 提升 IP 地址的分配效率
		- 一台主机接入网络后，只需向路由器确认本网络前缓，就可以给自己分配 IP 地址 (通常以主机自身的 MAC 地址作为接口标识符)
	- IPv6 **可以不使用 DHCP**。
		- 当然，在一些 IP 地址安全性要求高的地方，也支持使用 DHCP 统一管理 IPv6 地址
### 4.3.4 地址分类（三大类目的地址）

| 地址类型     | 二进制前缀<br>（最高有效位） | 标准记法        | 作用说明                                   |
| -------- | ---------------- | ----------- | -------------------------------------- |
| 未指明地址    | 全 0（128 位）       | `::/128`    | 表示“无地址”，类似 IPv4 的 `0.0.0.0`            |
| 环回地址     | 127 个 0 后跟 1     | `::1/128`   | 本机自环测试，类似 IPv4 的 `127.0.0.1` (自己和自己通信) |
| 多播地址     | `11111111`       | `FF00::/8`  | 数据报发往一组主机(类比群聊)                        |
| 本地链路单播地址 | `1111111010`     | `FE80::/10` | 局域网内通信，仅链路内有效，路由器不转发                   |
| 全球单播地址   | 除上述四种外的全部地址      | —           | 用于全球互联网可达的单播通信                         |
1. **单播（Unicast）**：数据报交付到**唯一**一台主机  
   - **全球单播地址**：除特殊预留外的所有地址，目前以 `2000::/3` 开头最常见  
   - **本地链路单播地址**：`FE80::/10` 开头，仅用于同一链路内通信，不可路由
2. **多播（Multicast）**：数据报同时交付到**一组**主机  
   - 固定前缀：`FF00::/8`（即高 8 bit 为 0xFF）
   - **应用**：网络会议
3. **任播（Anycast）**：数据报交付到**最近**（路由距离）的一台主机  
   - 没有固定前缀，通常由地址管理机构预先统一分配
   - **应用**：多个 DNS 服务器共享一个任播地址
### 4.3.5 地址资源与分配特点
- IPv6 **完全支持 CIDR**，取消“类”概念  
- 地址分配机构可按需灵活划分子网大小，如 `/32`、`/48`、`/56`、`/64` 等  
- 子网划分层次更多，利于聚合路由、降低全球路由表规模
- **地址空间巨大**：$2^{128}$ 约 $3.4×10^{38}$ 个地址，可划分更多层次  
- **层次化分配**：顶级（TLA）→ 次级（NLA）→ 站点级（SLA）→ 子网 → 链路
