### 1.3.1 电路交换
#### 1.3.1.1 简介
- **定义**：通过物理线路的连接，动态地分配传输线路资源
- **分为三个阶段**:
	- **建立连接**:建立一条专用的物理通路 (尝试占用通信资源) 
	- **通话**:主叫和被叫双方互相通电话 (一直占用通信资源) 
	- **释放连接**:释放刚才使用的专用的物理通路 (归还通信资源)
#### 1.3.1.2 特点
- **特点**：通信的两个用户**始终占用**端到端的通信资源
- **优点**：数据直接送达，传输速率高
- **缺点**：需要额外时间开销；独占导致利用率低；灵活性差；不支持差错控制
**使用场景**：适用于低频次大量传输数据
### 1.3.2 报文交换
#### 1.3.2.1 优点
- 通信前无需建立连接  
- 数据以“报文”为单位被交换节点间“存储转发”，通信线路可灵活分配；通信时间内两用户无需独占整条物理线路，线路利用率高于电路交换  
- 交换节点支持“差错控制”（通过校验技术）
#### 1.3.2.2 缺点
- 报文不定长，不便于存储转发管理  
- 长报文的存储转发时间开销大、缓存开销大  
- 长报文容易出错，重传代价高
### 1.3.3 分组交换
#### 1.3.3.1 简介
- **分组（packet）**：将数据划分成小段，每一段增加**首部**（表示发送的目标与数据的顺序）
- 路由器（分组交换机）**处理分组**的过程
	1. 暂存收到的分组  
	2. 检查分组首部  
	3. 查找转发表  
	4. 按照首部中的目的地址找到合适的接口转发出去  
- **转发**特点
	- 根据首部包含的目的地址、源地址等重要控制信息进行转发  
	- 每个分组在互联网中独立选择传输路径  
	- 位于网络核心部分的路由器负责转发分组，即进行分组交换  
	- 路由器需创建并动态维护转发表
#### 1.3.3.2 优缺点
- **优点**  ![[../../CN_Picture/Pasted image 20251026163206.png]]
  - 通信前无需建立连接  
  - 数据以“分组”为单位被交换节点间“**存储转发**”，通信线路可灵活分配；通信时间内两用户无需独占整条物理线路，线路利用率高于电路交换  
  - 交换节点支持“差错控制”（通过校验技术）  
- 相比报文交换的改进：分组定长，便于存储转发管理；存储转发时间与缓存开销小；分组不易出错，重传代价低  
- **缺点**  
  - 相比[[#1.3.2 报文交换|报文交换]]，控制信息占比增加  
  - 相比[[#1.3.1 电路交换|电路交换]]，存储转发时延依然存在
  - 报文被拆分为多个分组，传输过程中需增加处理复杂度
### 1.3.4 三种交换方式的性能对比
![[../../CN_Picture/Pasted image 20251026163326.png]]
上图中 ABCD 均为节点
若要连续传送大量的数据，且其传送时间远大于连接建立时间，则**电路交换**的传输速率较快。  
报文交换和分组交换**不需要**预先分配传输带宽，在传送突发数据时可提高整个网络的信道利用率。  
由于一个分组的长度往往远小于整个报文的长度，因此分组交换比报文交换的时延小，同时也具有更好的灵活性。

| 比较维度        | 电路交换            | 报文交换换 | 分组交换        |
| ----------- | --------------- | ----- | ----------- |
| 完成传输所需时间    | 最少(排除建立、释放连接耗时) | 最多    | 较少          |
| 存储转发时延      | 无               | 较高    | 较低          |
| 通信前是否需要建立连接 | 是               | 否     | 否           |
| 缓存开销        | 无               | 高     | 低           |
| 是否支持差错控制    | 不支持             | 支持    | 支持          |
| 报文数据有序到达    | 是               | 是     | 否           |
| 是否需要额外的控制信息 | 否               | 是     | 是(控制信息占比最大) |
| 线路分配灵活性     | 不灵活             | 灵活    | 非常灵活        |
| 线路利用率       | 低               | 高     | 非常高         |
